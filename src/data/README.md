# 车辆品牌型号数据

本文件包含系统中使用的所有车辆品牌和型号数据。

## 数据结构

```json
{
  "brands": [
    "品牌 型号",
    ...
  ],
  "lastUpdated": "YYYY-MM-DD",
  "version": "1.0.0"
}
```

## 如何更新车型数据

### 方法 1：直接编辑 JSON 文件

1. 打开 `src/data/vehicle-brands.json` 文件
2. 在 `brands` 数组中添加或删除车型
3. 更新 `lastUpdated` 日期
4. 更新 `version` 版本号
5. 保存文件，系统会自动更新

### 方法 2：使用文本编辑器批量处理

1. 从其他数据源（如 Excel、CSV）复制车型列表
2. 使用文本编辑器的查找替换功能格式化数据
3. 粘贴到 `brands` 数组中
4. 确保每个车型都用双引号包裹，并用逗号分隔

### 格式要求

- 每个车型占一行
- 使用双引号包裹车型名称
- 最后一个车型后面不要加逗号
- 车型名称格式：`品牌 型号`（例如：`比亚迪 汉 EV`）

## 数据分类建议

为了便于管理，建议按以下分类组织车型（使用注释分隔）：

```json
{
  "brands": [
    // ==================== 特斯拉 ====================
    "特斯拉 Model 3",
    "特斯拉 Model S",
    
    // ==================== 比亚迪 ====================
    "比亚迪 汉 EV",
    "比亚迪 唐 EV",
    
    // ... 其他品牌
  ]
}
```

## 数据来源

- 汽车品牌官网
- 汽车之家、易车网等汽车网站
- 工信部新车公告目录
- 用户反馈和建议

## 更新频率

建议每季度更新一次，或者：
- 当有新车发布时及时添加
- 当用户反馈找不到某款车型时添加
- 定期清理已停产的车型

## 注意事项

1. 确保 JSON 格式正确（可以使用 JSON 验证工具）
2. 车型名称要准确、规范
3. 避免重复的车型
4. 保持文件大小合理（目前约 500+ 款车型）

## 示例

添加新车型的完整流程：

1. 打开 `vehicle-brands.json`
2. 在合适的位置添加新车型：
   ```json
   "小米 SU7",
   "小米 SU7 Pro",
   "小米 SU7 Max",
   ```
3. 更新元数据：
   ```json
   "lastUpdated": "2026-01-10",
   "version": "1.0.1"
   ```
4. 保存文件
5. 重新编译或刷新页面即可看到新车型
