根据《2024年湖南省物联网应用创新设计竞赛》任务书的要求，城市智能电车管理与换电系统是一个典型的物联网+云边协同+数据可视化+路径规划的综合项目。下面我为你提供一个完整的系统设计方案，包括前后端架构、功能模块划分、技术选型建议，以及可扩展的企业级功能，使该项目不仅满足竞赛要求，还能具备实际落地价值。

✅ 一、系统总体架构
1. 系统分层设计（三层架构）
层级	说明	技术选型建议
感知层	电车上的硬件设备：传感器、RFID、控制器、显示屏、蜂鸣器、按键等	Arduino/ESP32/STM32 + 各类传感器
网络层	数据上传与指令下发：Wi-Fi、MQTT/HTTP、局域网	MQTT（推荐）、HTTP/RESTful API
应用层	后端服务 + 管理前端（Web/APP）	Spring Boot / Node.js + Vue / React / Flutter
✅ 二、功能模块设计
1. 车载系统（嵌入式端）
基础功能：
传感器数据采集（电压、温度、光线）

RFID读取电池编号

OLED/LCD显示车辆信息（VID、PID、电压、温度、电量、灯光状态）

按键控制喇叭

自动灯光控制（基于光线传感器 + 对向来车逻辑）

异常报警（蜂鸣器 + 上传报警类型）

接收服务器指令（闪灯、鸣笛、推送信息）

扩展功能（企业级）：
OTA远程升级：支持固件远程更新

本地缓存：网络中断时暂存数据，恢复后补传

GPS模拟模块：使用按键/电位器模拟移动和位置变化

驾驶行为记录：记录急刹车、急加速（可选加速度传感器）

2. 后端服务（核心业务逻辑）
基础功能：
设备注册与管理（VID唯一性）

数据接收与存储（MySQL + InfluxDB 时序数据库）

车辆状态实时更新（WebSocket / MQTT）

电池历史数据记录（电压、温度、电量变化）

异常报警处理与推送

路径规划算法（BFS/最短路径）

换电推荐逻辑（基于剩余电量和距离）

扩展功能（企业级）：
设备影子（Device Shadow）：保存设备最新状态，支持断线重连后同步

规则引擎：自定义报警规则（如温度 > 60℃触发告警）

消息队列（Kafka/RabbitMQ）：处理高并发数据上传

用户管理：多角色权限（管理员、运维、司机）

日志审计：所有操作记录，便于追溯

API网关：统一入口，限流、鉴权、监控

3. 管理系统前端（Web/APP）
基础功能：
车辆列表与状态展示（在线/离线、实时数据）

地图展示（车辆位置、换电站、障碍物）

车辆位置手动设置（坐标输入/点选）

车辆远程控制（闪灯、鸣笛）

电池历史数据图表（ECharts/Highcharts）

异常信息展示（时间轴形式）

电量不足提醒 + 最短路径展示

换电站推荐与路径规划

扩展功能（企业级）：
大屏可视化：用于监控中心展示

历史轨迹回放：记录车辆移动路径

车辆调度看板：展示任务、状态、电量、建议换电时间

消息推送中心：支持短信/邮件/APP推送

地图热力图：展示车辆密度、换电需求区域

车辆健康评分：基于电池寿命、异常次数等综合评分

✅ 三、数据库设计建议
主要表结构（MySQL）
表名	字段说明
vehicle	VID、状态、当前坐标、最后在线时间、当前电池PID
battery	PID、Vmin、Vmax、当前状态（使用中/空闲）
battery_history	PID、VID、电压、温度、电量、时间戳
alert_log	异常类型、时间、VID、处理状态
station	换电站ID、坐标、电池库存
command_log	指令类型、目标VID、发送时间、状态
时序数据库（InfluxDB）
存储电池电压、温度、电量的连续变化数据

用于图表展示和趋势分析
